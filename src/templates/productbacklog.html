Product Backlog

O Product Backlog é uma lista de requisitos do cliente mantida pelo Product Owner. Pode ser alterada a qualquer momento. Durante o Sprint Planning, a equipe escolhe itens do topo do backlog para trabalhar na sprint, priorizando e planejando sua execução.

  • Identificação de Itens do Backlog: Coleta de ideias, requisitos e demandas do produto de várias fontes, incluindo stakeholders, usuários, equipes de desenvolvimento e feedbacks.
  • Priorização: Priorize os itens do Product Backlog com base em importância e valor, utilizando técnicas como a matriz de valor x esforço para guiar as decisões.
  • Detalhamento e Refinamento: A medida que os itens sobem na lista de prioridades, detalhe e refine-os. Isso inclui dividir tarefas maiores em mais simples e garantir clareza na equipe sobre o que deve ser feito.
  • Estimativa: Priorize e detalhe os itens do backlog. Atribua estimativas de esforço para entender o trabalho envolvido.
  • Manutenção Contínua: O Product Backlog é uma lista dinâmica, sujeita a revisões e atualizações constantes conforme novas informações surgem e as prioridades mudam.
  • Comunicação e Colaboração: Comunique-se regularmente para manter o Product Backlog alinhado com as necessidades do produto, incluindo reuniões de refinamento.

    Criando um Backlog:

    Crie uma tabela com os seguintes itens:

  1. Epic: Aqui será onde você irá definir um objetivo principal da Sprint (MVP).
  2. User Story: Liste todas as funcionalidades que você deseja para o seu site e organize as User Stories por ordem de importância.
    Exemplo: Como um usuário, eu quero poder criar uma nova tarefa para que eu possa organizar meu trabalho diário de forma eficaz.
  3. Sprint: Coloque o número da sprint relacionada.
  4. Prioridade: Defina as prioridades utilizando números de 0 a 100.
    5. Exemplos:
    • Tarefa 1: Corrigir bug crítico no sistema (Prioridade: 90).
    • Tarefa 2: Implementar nova funcionalidade solicitada pelo cliente (Prioridade: 85).
    • Tarefa 3: Melhorar a interface do usuário para aumentar a usabilidade (Prioridade: 70).
  5. Ferramentas: Coloque as ferramentas utilizadas que serão usadas para realizar uma tarefa específica.
    Exemplo: Utilizar o Jira para criar um quadro Kanban, movendo as tarefas concluídas para a coluna "Concluído" e mantendo as tarefas em andamento na coluna "Em Progresso".
