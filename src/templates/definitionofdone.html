Definition of Done

A "Definition of Done" (DoD) é um conceito fundamental no framework Scrum que estabelece os critérios que um incremento de produto deve atender para ser considerado completo e pronto para ser entregue ao cliente ou liberado em produção. A DoD é uma ferramenta essencial para garantir que a equipe de desenvolvimento entenda o que significa "feito" em um contexto específico de projeto e que todos os itens de backlog atendam a esses critérios antes de serem considerados concluídos.

Geralmente, a DoD inclui e checa os seguintes aspectos:

  • Funcionalidade: A funcionalidade implementada deve estar completa e funcionando conforme as especificações.
    Por exemplo, se a equipe estiver construindo um sistema de registro de usuários, a funcionalidade estará pronta quando os usuários puderem se registrar, receber confirmações do registro e fazer login com sucesso.
  • Testes: Todas as funcionalidades devem ser testadas e passar nos testes de qualidade estabelecidos para a tarefa. Isso pode incluir unit tests, testes de integração, aceitação do usuário, e outros aspectos definidos pelo time.
  • Design: A tarefa deve seguir as diretrizes de design estabelecidas, como a aparência, usabilidade e consistência visual.
  • Documentação: Qualquer documentação relevante, como manuais de usuário, documentação técnica ou notas de versão, foi atualizada para refletir as mudanças feitas no incremento. Isso ajuda na compreensão e na utilização adequada do produto.
  • Desempenho: A tarefa deve desempenhar de forma conforme os requisitos estabelecidos.
    Por exemplo, um sistema de registro de usuários deve responder de maneira rápida e eficiente sob carga.
  • Revisão de código: O código-fonte deve ser revisado pela equipe e preferencialmente, ter um formato e utilizar convenções consistentes com outros códigos ou normas já presentes.

Exemplos:

  • Para uma User Story de um sistema de gerenciamento de tarefas, sua definição de pronto pode incluir critérios como:
    1. A User Story foi implementada.
    2. Passou nos testes de unidade.
    3. Testes de aceitação do usuário foram realizados.
    4. A documentação foi atualizada.
  • Para uma sprint em um projeto, a DoD pode incluir os seguintes critérios:
    1. Todas as User Stories foram realizadas.
    2. Todas as User Stories foram implementadas.
    3. Todas as User Stories foram revisadas.
    4. Todas as User Stories foram documentadas.
  • Para o lançamento de um novo site, a DoD pode incluir:
    1. Todas as páginas foram criadas com base no design aprovado.
    2. Todos os links estão funcionando.
    3. O site foi testado em diferentes navegadores e dispositivos.
    4. A documentação de suporte está pronta.

Somente quando todos os critérios acima são cumpridos, uma tarefa deve ser considerada finalizada, e seu valor incrementado ao Backlog. Caso uma tarefa não cumpra com os requisitos, portanto não sendo considerada finalizada, seu valor não deve ser incrementado ao Backlog e a mesma não deve ser apresentada ao cliente ou ser inclusa na Sprint Review.
